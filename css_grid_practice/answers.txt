1. Основное свойство CSS Grid и отличие от Flexbox
Основное свойство CSS Grid – это двумерная система компоновки, которая позволяет работать одновременно и по строкам, и по столбцам. Вы создаете четкую структуру сетки, а затем размещаете элементы в любом месте этой сетки. Принципиальное отличие от Flexbox в том, что Flexbox работает только в одном направлении – либо как строка, либо как колонка. Если вам нужно управлять расположением и по горизонтали, и по вертикали одновременно, Grid справляется с этим естественно, а Flexbox требует вложенных контейнеров.

 2. Разница между grid-template-columns и grid-auto-columns
grid-template-columns создает явную сетку – вы заранее определяете, сколько будет столбцов и каких размеров. Например, grid-template-columns: 100px 1fr 200px создаст три конкретных столбца.
grid-auto-columns работает с неявной сеткой – он определяет размер для любых дополнительных столбцов, которые могут появиться, если элементы выйдут за пределы явно заданной сетки. Например, если вы разместите элемент в пятом столбце, но определили только три явных столбца, grid-auto-columns задаст размер для этих новых, автоматически созданных столбцов.

 3. Функции repeat(), minmax() и auto-fit
repeat() позволяет повторять шаблоны столбцов или строк. Вместо того чтобы писать 1fr 1fr 1fr 1fr, можно написать repeat(4, 1fr).
minmax() задает минимальный и максимальный размер для трека. Например, minmax(200px, 1fr) означает, что столбец будет не меньше 200 пикселей, но может расширяться до доступного пространства.
auto-fit автоматически подбирает количество столбцов, чтобы заполнить все доступное пространство. В комбинации с minmax это создает идеально адаптивные сетки: repeat(auto-fit, minmax(250px, 1fr)) создаст столбцы минимум по 250px, которые будут растягиваться и переноситься на новую строку при нехватке места.

 4. Именованные grid-области
Именованные области позволяют визуально описывать макет с помощью свойства grid-template-areas. Вы даете именам различным зонам макета (header, sidebar, main, footer) и затем просто присваиваете эти имена элементам через grid-area.

Преимущества: невероятная наглядность – глядя на CSS, вы сразу видите всю структуру макета. Легко перестраивать макет, просто меняя схему в grid-template-areas. Отлично работает с медиа-запросами для адаптивности.

 5. Система нумерации grid-линий и размещение элементов
Сетка состоит из линий, которые нумеруются с 1 (с начала) и с -1 (с конца). grid-column и grid-row позволяют размещать элементы, указывая между какими линиями они должны находиться.
Например, grid-column: 1 / 3 разместит элемент от первой до третьей линии (занимая два столбца). grid-row: 2 / span 2 разместит элемент со второй линии, занимая две строки.

 6. grid-gap (или gap) vs margins
gap создает отступы именно между элементами сетки, не затрагивая внешние края. Это лучше, чем margins, потому что:
- Отступы задаются одним свойством для всей сетки
- Нет проблем со схлопыванием margin'ов
- Не нужно обнулять margins у крайних элементов
- Отступы выглядят предсказуемо и аккуратно

 7. justify-items и align-items
justify-items выравнивает элементы по горизонтали внутри их ячеек сетки
align-items выравнивает элементы по вертикали внутри их ячеек сетки
Они работают с содержимым внутри грид-областей, а не с самой сеткой в целом. Значения типа start, end, center, stretch позволяют контролировать позиционирование content'a внутри каждой ячейки.

 8. Разница между auto-fill и auto-fit
Разница проявляется, когда есть свободное место в строке. auto-fill создает максимально возможное количество треков (включая пустые), чтобы заполнить строку. auto-fit растягивает существующие элементы, чтобы занять все доступное пространство, схлопывая пустые треки. Проще говоря: auto-fill оставляет место для будущих элементов, auto-fit растягивает то, что есть.

 9. Создание сложных макетов с grid-template-areas
С помощью grid-template-areas вы можете создавать сложные макеты, визуально описывая их в CSS. Вы просто рисуете схему расположения областей в виде текста, где каждая строка представляет собой строку сетки, а каждое слово – область.
Правила: использовать только одно слово для имени области, одинаковое количество областей в каждой строке, точки для пустых областей. Области должны быть прямоугольными - нельзя создавать Г-образные формы.

 10. Преимущества CSS Grid для сложных двумерных макетов
Grid предоставляет беспрецедентный контроль над двумерными макетами. Вы можете легко создавать сложные расположения, которые раньше требовали хаков или JavaScript. Код становится чище и семантичнее, макеты - более гибкими и адаптивными. Это единственная система в CSS, которая действительно предназначена для работы с двумерными пространствами, что делает ее идеальной для современных веб-интерфейсов.
